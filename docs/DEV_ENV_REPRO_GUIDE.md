# React_LostArk 개발 환경 재현 가이드 (AI/MCP/Plugin 포함)

이 문서는 **다른 PC/세션에서도 현재 작업 환경을 최대한 동일하게 재현**하기 위한 가이드입니다.

## 1) 기본 로컬 개발 환경

### 필수 버전
- Node.js 20 LTS 권장
- Yarn 1.22.x
- Git 최신 버전
- (Windows 기준) PowerShell 5.1 이상

### 프로젝트 실행
```bash
yarn install
yarn start
```
- 기본 접속: `http://localhost:3000`
- 빌드 검증: `yarn build`

---

## 2) AI 작업 환경 (OpenCode 계열)

이 저장소는 AI 에이전트 기반 워크플로우를 사용합니다. 아래 항목이 동일해야 작업 품질이 일관됩니다.

### 핵심 설정/디렉터리
- 프로젝트 루트: `React_LostArk`
- 워크플로우 문서: `.agent/workflows/`
- 전역 로컬 규칙 참조: `C:\Users\한동건\.claude\CLAUDE.md`

### 권장 확인 항목
- 에이전트가 프로젝트 루트를 정확히 인식하는지
- `.agent/workflows/review-local-code.md` 같은 워크플로우 파일이 존재하는지
- 로컬 규칙 파일 경로가 현재 PC에서도 유효한지

---

## 3) MCP/도구 세팅 체크리스트

> 주의: MCP 서버/키는 개인 환경값이므로, 실제 토큰/시크릿은 저장소에 커밋하지 마세요.

### 필수 원칙
- API 키/토큰은 OS 환경변수 또는 개인 시크릿 스토어에 저장
- 프로젝트 파일에는 샘플 키만 문서화 (`.env.example` 형태)
- 실제 키는 절대 Git 추적 금지

### 최소 점검 항목
- 브라우저 자동화 MCP 연결 가능 여부
- GitHub MCP 인증 상태
- 검색/문서 조회 MCP 인증 상태
- SQL/외부 데이터 MCP를 쓴다면 연결 문자열 분리 여부

### 빠른 검증 방법(예시)
1. 에이전트 세션에서 MCP 목록 조회
2. 브라우저 페이지 열기 테스트 1회
3. GitHub 조회 API 1회
4. 실패 시 인증/권한 범위(scope) 확인

---

## 4) Plugin/Skill 재현 포인트

현재 작업에서는 커스텀 워크플로우 명령(예: `/review/local`)을 사용합니다.

### 확인해야 할 것
- 해당 명령이 참조하는 워크플로우 파일 존재 여부
- 플러그인/스킬 로딩 순서가 기존 환경과 동일한지
- 프로젝트/전역 규칙 충돌 여부

### 문제 발생 시 우선 점검
- 경로 오타 (`.agent/...`)
- 명령명/슬래시 명령 버전 불일치
- 최신 플러그인 업데이트로 인한 스키마 변경

---

## 5) 환경변수/시크릿 관리 권장안

- 로컬: 시스템 환경변수 또는 사용자 시크릿 매니저
- 팀 공유: 키 자체 공유 금지, 발급/권한 가이드만 공유
- 저장소: `.env.example`만 유지, `.env`는 `.gitignore` 유지

권장 변수 분리 예시:
- `LOSTARK_API_KEY` - Vercel 함수에서만 읽는 서버 전용 Lost Ark API 키
- `GITHUB_TOKEN`
- `MCP_*`

---

## 6) 재현 완료 체크리스트 (Done 정의)

아래가 모두 통과되면 “현재와 동일한 작업 환경”으로 간주:

- [ ] `yarn install` 성공
- [ ] `yarn start` 후 앱 정상 접속
- [ ] `yarn build` 성공
- [ ] AI 에이전트가 프로젝트 규칙/워크플로우 파일 인식
- [ ] 주요 MCP 1회 이상 호출 성공
- [ ] `/review/local` 같은 핵심 명령 실행 가능

---

## 7) 트러블슈팅

### A. 빌드는 되는데 MCP가 실패
- 인증 토큰 만료/누락 가능성 높음
- 권한(scope) 또는 네트워크 정책 점검

### B. 슬래시 명령은 있는데 동작이 다름
- 워크플로우 파일 버전 차이 확인
- 전역 규칙 파일 경로/내용 불일치 점검

### C. AI 출력 톤/행동이 다름
- 세션별 시스템/개발자 지시문 차이 확인
- 프로젝트 내 규칙 파일(AGENTS/워크플로우) 로딩 여부 점검

---

## 8) 권장 운영 방식

- 새 환경 세팅 후 첫 작업 전:
  1) 이 문서 체크리스트 수행
  2) `/review/local`로 작은 변경 1건 리뷰 테스트
  3) 결과가 기존 환경과 유사하면 본 작업 진행

필요하면 다음 단계로, 이 문서를 기반으로 **환경 부트스트랩 스크립트(Windows/macOS 분리)**까지 추가할 수 있습니다.
